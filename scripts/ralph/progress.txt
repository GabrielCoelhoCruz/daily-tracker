## Codebase Patterns
- Expo SDK 54 with Expo Router file-based routing (`app/` directory)
- NativeWind v4 with Tailwind CSS for styling (`className` prop on RN components)
- Import global.css in root `app/_layout.tsx`
- Custom Tailwind colors in `tailwind.config.js`: bg-primary (#0c0a09), bg-card (#1c1917), bg-elevated (#292524), accent (#f59e0b), txt-primary (#fafaf9), txt-secondary (#a8a29e), txt-muted (#78716c)
- `constants/theme.ts` exports `theme` object with all color tokens — use for inline styles when NativeWind classes aren't sufficient
- `@/` path alias for project root imports
- `npx tsc --noEmit` for typecheck
- Zustand stores in `stores/` with persist middleware + AsyncStorage
- Access stores outside React: `useStore.getState()` / `useStore.subscribe()`
- NativeWind `className` for static layout styles, inline `style` for dynamic colors and typography
- expo-haptics already used in CheckItem, HidratacaoCard, CardioCard, index.tsx (Light impact)
- Current tab bar: custom GlassTabBar in `app/(tabs)/_layout.tsx` using `expo-glass-effect` + JS `Tabs`
- Current icons: Ionicons (12+ files) + FontAwesome (tab bar only) — both from `@expo/vector-icons`
- Current route structure: flat `app/(tabs)/index.tsx`, `treino.tsx`, `historico.tsx`
- DiaDetalhe currently uses React Native `<Modal>` with `animationType="slide"` inside historico screen
- react-native-reanimated installed but NOT used anywhere
- expo-blur installed but NOT imported anywhere
- SDK 54 NativeTabs: use standalone `Icon`, `Label`, `VectorIcon` imports (NOT compound `NativeTabs.Trigger.X`)
- SDK 54 NativeTabs: `md` prop NOT available — use `VectorIcon` with MaterialCommunityIcons for Android fallback
- NativeTabs don't render headers — must nest Stack inside each tab group
- ScrollView must be first opaque child of each screen for tab bar transparency on iOS 18
- formSheet presentation requires a route in a Stack — cannot be used with in-screen `<Modal>`
