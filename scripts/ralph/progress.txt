## Codebase Patterns
- Expo SDK 55 with Expo Router file-based routing (`app/` directory)
- NativeWind v4 with Tailwind CSS for styling (`className` prop on RN components)
- Custom Tailwind colors in `tailwind.config.js`: bg-primary (#0c0a09), bg-card (#1c1917), bg-elevated (#292524), accent (#f59e0b), txt-primary (#fafaf9), txt-secondary (#a8a29e), txt-muted (#78716c)
- `constants/theme.ts` exports `theme` object with all color tokens + `typography` (caption, footnote, body, callout, headline, title3) â€” use for inline styles
- `@/` path alias for project root imports
- `npx tsc --noEmit` for typecheck
- Zustand stores in `stores/` with persist middleware + AsyncStorage â€” see `useConfigStore.ts` for pattern
- NativeWind `className` for static layout styles, inline `style` for dynamic colors and typography
- Tab bar in `app/(tabs)/_layout.tsx` uses regular `Tabs` from expo-router with `Ionicons` for tab icons
- All other components use `MaterialCommunityIcons` from `@expo/vector-icons/MaterialCommunityIcons` (default import, NOT named import)
- Route structure: `app/(tabs)/(hoje)/index.tsx`, `(treino)/index.tsx`, `(historico)/index.tsx`, `(historico)/dia-detalhe.tsx`
- Sub-routes use formSheet presentation: `presentation: 'formSheet'`, `sheetGrabberVisible: true`, `sheetAllowedDetents: [0.5, 1.0]`
- All screens use ScrollView as first child with `contentInsetAdjustmentBehavior="automatic"`
- `<Card>` component in `components/ui/Card.tsx` â€” always use for card-style containers
- `<Card className="p-0">` when component manages own internal padding
- Screen content gaps use `gap-3` (12px) between cards
- Settings/nav buttons go in `headerRight` of Stack.Screen options
- Use `minHeight: 44` via inline `style` for touch target compliance
- `npm install --legacy-peer-deps` required due to react-native-web peer dependency conflict with React 19
- `process.env.EXPO_OS === "ios"` for iOS-only code (not Platform.OS)
- `borderCurve: 'continuous'` for iOS squircle corners â€” must be inline style, not NativeWind
- expo-file-system SDK 55: use `File`/`Directory`/`Paths` from root import; legacy API at `/legacy` throws at runtime
- Use `file.base64()` instead of `readAsStringAsync` with `EncodingType.Base64`
- Typed routes: use `as any` cast for new route groups not yet in generated types
