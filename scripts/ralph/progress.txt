## Codebase Patterns
- Expo SDK 55 with Expo Router file-based routing (`app/` directory)
- NativeWind v4 with Tailwind CSS 3.3.2 for styling (`className` prop on RN components)
- Import global.css in root `app/_layout.tsx`
- Custom Tailwind colors defined in `tailwind.config.js` under `theme.extend.colors`: bg-primary (#0c0a09), bg-card (#1c1917), bg-elevated (#292524), accent (#f59e0b), txt-primary (#fafaf9), txt-secondary (#a8a29e), txt-muted (#78716c)
- Tab bar uses inline styles (not NativeWind) for backgroundColor/tint colors in `app/(tabs)/_layout.tsx`
- `app/config.tsx` is a modal route (presentation: 'modal' in root Stack)
- TypeScript strict mode enabled via `tsconfig.json` extending `expo/tsconfig.base`
- Use `@/` path alias for project root imports
- `npx tsc --noEmit` for typecheck
- When creating new Expo project alongside existing files, use temp dir approach then copy files over
- `constants/theme.ts` exports a `theme` object with all color tokens (bg, accent, semantic, text, border) — use for inline styles when NativeWind classes aren't sufficient
- Shared UI components in `components/ui/` (Card, Badge) — use NativeWind classes + inline `style` for dynamic colors
- Checklist components in `components/checklist/` (ProgressBar) — accept typed props like `completados`/`total`
- Badge uses `color + "20"` hex alpha for background tint effect

  - For dynamic colors (like Badge background based on prop), use inline `style` prop alongside NativeWind `className`
  - Hex alpha shorthand (e.g. `"#f59e0b20"`) works for semi-transparent backgrounds in RN
  - The `border` Tailwind color class maps to `border-border` (since the key is `border.DEFAULT` in tailwind.config.js)
