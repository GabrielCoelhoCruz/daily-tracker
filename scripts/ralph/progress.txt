## Codebase Patterns
- Expo SDK 55 with Expo Router file-based routing (`app/` directory)
- NativeWind v4 with Tailwind CSS 3.3.2 for styling (`className` prop on RN components)
- Import global.css in root `app/_layout.tsx`
- Custom Tailwind colors defined in `tailwind.config.js` under `theme.extend.colors`: bg-primary (#0c0a09), bg-card (#1c1917), bg-elevated (#292524), accent (#f59e0b), txt-primary (#fafaf9), txt-secondary (#a8a29e), txt-muted (#78716c)
- Tab bar uses inline styles (not NativeWind) for backgroundColor/tint colors in `app/(tabs)/_layout.tsx`
- `app/config.tsx` is a modal route (presentation: 'modal' in root Stack)
- TypeScript strict mode enabled via `tsconfig.json` extending `expo/tsconfig.base`
- Use `@/` path alias for project root imports
- `npx tsc --noEmit` for typecheck
- When creating new Expo project alongside existing files, use temp dir approach then copy files over
- `constants/theme.ts` exports a `theme` object with all color tokens (bg, accent, semantic, text, border) — use for inline styles when NativeWind classes aren't sufficient
- Shared UI components in `components/ui/` (Card, Badge) — use NativeWind classes + inline `style` for dynamic colors
- Checklist components in `components/checklist/` (ProgressBar) — accept typed props like `completados`/`total`
- Badge uses `color + "20"` hex alpha for background tint effect
- Plan data lives in `data/plano.ts` — exports `plano` constant and all types (Categoria, RegraCondicional, ItemDoPlano, Periodo, Dica, Plano)
- Items with `subItens` use a parent "Suplementos" item with the sub-items array for grouped supplements
- Conditional rules use `regra: { diasDaSemana: [1,5] }` for day-specific items and `regra: { apenasEmDiaDeTreino: true }` for training-day-only items
- Period-level `regra` (e.g. Intra Treino) applies to the entire period; item-level `regra` applies to individual items
- Optional items use `opcional: true` flag (Abdômen, Panturrilha in "extras" period)
- Workout data lives in `data/treinos.ts` — exports `treinos` array and types (Serie, Exercicio, Treino)
- Serie type uses `tipo: "WS" | "TS" | "BS" | "CS"` with `series` count and optional `reps` (number or string like "4+4+4")
- Treino has `letra` (A-E) and `grupoMuscular` for display; exercises have `observacao` for special notes like "1seg pico de contração"
- Tips data lives in `data/dicas.ts` — imports `Dica` type from `data/plano.ts`, exports `dicas` array with 25 tips (12 nutricao + 13 treino)
- Day utilities in `utils/diaUtils.ts` — `isDiaDeTreino`, `getTreinoDoDia`, `filtrarItensDoDia`, `contarItens`. Uses JS dayOfWeek (0=Sun, 1=Mon...6=Sat). Mon-Fri are training days unless `diaOffManual` is true.
- Item counting: parent items with `subItens` are NOT counted — only their sub-items are counted individually
- History store in `stores/useHistoryStore.ts` — exports `HistoricoDia` type and `useHistoryStore`. Uses `get()` for read-only actions (getDia)
- For Zustand read-only/getter actions, use `get()` instead of `set()` in the store creator
- All 3 Zustand stores: `useDayStore` (day-store), `useHistoryStore` (history-store), `useConfigStore` (config-store) — all use persist middleware with AsyncStorage
- Access Zustand stores outside React: `useDayStore.getState()` / `useDayStore.setState({})` — no hooks needed in utility functions
- Reset logic in `utils/resetUtils.ts` — `checkAndReset()` uses 4am day boundary; call on app mount to handle day transitions
- Use `new Date(dateStr + "T12:00:00")` to avoid timezone/DST issues when parsing YYYY-MM-DD strings
- `expo-haptics` available for haptic feedback — use `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)` for taps
- Icons from `@expo/vector-icons` (Ionicons) — pre-installed with Expo, no extra install needed
- Items with `subItens` are group headers (not checkable) — only render checkmarks on leaf items
- Cardio component in `components/cardio/CardioCard.tsx` — uses `plano.metaCardioMin` (80min) for goal; sessions stored as `SessaoCardio[]` with `minutos` + `timestamp`
- Dicas component in `components/dicas/DicasSection.tsx` — accepts `categoria` prop ('nutricao' | 'treino'), filters tips from `data/dicas.ts`, collapsed by default. Training section includes series legend (WS/TS/BS/CS)
- Tab Hoje screen (`app/(tabs)/index.tsx`) assembles all components: ProgressBar, PeriodoSection (filtered), HidratacaoCard, CardioCard, DicasSection. Uses `contentContainerClassName` for ScrollView content styling
- `useRouter` from expo-router for programmatic navigation (e.g. `router.push("/config")`)
- Refeição livre button only shown on periods with `refeicao` category items — filter with `periodo.itens.some(i => i.categoria === "refeicao")`
- Treino components in `components/treino/` (ExercicioItem) — read-only, no store deps, just renders from `data/treinos.ts`
