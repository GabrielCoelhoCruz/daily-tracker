## Codebase Patterns
- Expo SDK 54 with Expo Router file-based routing (`app/` directory)
- NativeWind v4 with Tailwind CSS for styling (`className` prop on RN components)
- Import global.css in root `app/_layout.tsx`

- Zustand stores in `stores/` with persist middleware + AsyncStorage
- Access stores outside React: `useStore.getState()` / `useStore.subscribe()`
- NativeWind `className` for static layout styles, inline `style` for dynamic colors and typography
- expo-haptics already used in CheckItem, HidratacaoCard, CardioCard, index.tsx (Light impact)
- Tab bar: NativeTabs from `expo-router/unstable-native-tabs` with SF Symbols via `Icon` component
- All icons now use `MaterialCommunityIcons` from `@expo/vector-icons/MaterialCommunityIcons` (default import)
- Import style: `import MaterialCommunityIcons from "@expo/vector-icons/MaterialCommunityIcons"` (NOT named import)
- Route structure: `app/(tabs)/(hoje)/index.tsx`, `(treino)/index.tsx`, `(historico)/index.tsx`, `(historico)/dia-detalhe.tsx`
- All screens use ScrollView as first child with `contentInsetAdjustmentBehavior="automatic"`
- react-native-reanimated installed but NOT used anywhere
- expo-blur installed but NOT imported anywhere
- SDK 54 NativeTabs: use standalone `Icon`, `Label`, `VectorIcon` imports (NOT compound `NativeTabs.Trigger.X`)
- SDK 54 NativeTabs: `md` prop NOT available â€” use `VectorIcon` with MaterialCommunityIcons for Android fallback
- NativeTabs don't render headers â€” must nest Stack inside each tab group
- ScrollView must be first opaque child of each screen for tab bar transparency on iOS 18
- formSheet presentation requires a route in a Stack â€” cannot be used with in-screen `<Modal>`
- Route structure now: `app/(tabs)/(hoje)/index.tsx`, `(treino)/index.tsx`, `(historico)/index.tsx`, `(historico)/dia-detalhe.tsx`
- Tab group names in Tabs.Screen must match directory names exactly: `(hoje)`, `(treino)`, `(historico)`
- ThemeProvider with DarkTheme wraps root Stack in `app/_layout.tsx`
- Typed routes auto-generate from file structure â€” run `npx expo customize tsconfig.json` to regenerate `.expo/types/router.d.ts`
- For typed routes: use `/dia-detalhe?date=...` (short form) not `/(historico)/dia-detalhe?date=...`
- `theme.typography` has pre-built TextStyle objects (caption, footnote, body, callout, headline, title3) â€” use via `style={theme.typography.callout}` for consistent text hierarchy
- Add `fontVariant: ['tabular-nums']` to any numeric display that changes dynamically (prevents layout jumps)
- Add `selectable` prop to Text elements showing user data (stats, percentages, counts)
- Always use `<Card>` component for card-style containers â€” never duplicate `cardShadow`/border styles inline
- `<Card className="p-0">` when the component manages its own internal padding (e.g., collapsible sections with header + content areas)
- Card has: borderRadius 16, borderCurve 'continuous', iOS shadow + Android elevation, bg-bg-card, p-4 default padding
- Screen content gaps use `gap-3` (12px) between cards
- Settings/nav buttons go in `headerRight` of Stack.Screen options, not inline in screen JSX
- Use `minHeight: 44` via inline `style` for touch target compliance on interactive elements
- CheckItem uses `checkbox-marked-circle-outline` / `checkbox-blank-circle-outline` from MaterialCommunityIcons
- SerieBadge/pill badges: use inline `style` with `borderCurve: 'continuous'`, `borderRadius: 6` â€” NativeWind `rounded` doesn't set borderCurve
- Circular number badges: `width`/`height` + `borderRadius: half` + accent color at `"20"` opacity suffix for background
- ExercicioItem uses `<Card className="p-0">` with internal p-3 padding sections
